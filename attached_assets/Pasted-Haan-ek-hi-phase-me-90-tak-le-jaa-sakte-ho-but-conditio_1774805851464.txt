# PHASE: Production Integration Lock

## 1) Flower + Roll tooling payload ko complete wire karo

Abhi sabse bada score killer yehi hai.

* `thicknessBandMin`
* `thicknessBandMax`
* `profileSourceType`

### Karna kya hai

* flower generation API payload update
* roll tooling API payload update
* backend schema update

* response metadata update

### Result

Band cosmetic nahi rahega.
Roll gap, pass planning, conservative tooling logic me use hoga.

### Expected score jump

Bahut bada. Yeh single fix hi major bucket recover karega.

---

## 2) Profile source offset ko geometry pipeline me actual apply karo

Abhi selector hai, but geometry unchanged hai.
Yani feature aadha hai.

### Rules

* `centerline` → no offset
* `inner_face` → outward offset by thickness
* `outer_face` → inward offset by thickness
* agar closed/open contour logic alag hai to deterministic side handling ho

### Karna kya hai

* profile normalization layer banao
* source-type resolved geometry banao
* flower generator ko raw geometry nahi, normalized geometry do
* preview me before/after show karo

### Result

Source selector real effect dega.

### Expected score jump

Strong P0 recovery.

---

## 3) Python API routing ko fully fix karo

`/health` 404 production trust tod deta hai.

### Karna kya hai

* router prefix normalize karo
* `/api/v1/health` aur expected route paths align karo
* frontend se ek real health probe add karo
* startup pe Python availability status show karo
* fail ho to silent success na do

### Minimum success condition

* health route 200 return करे
* at least 1 engineering path Node → Python → response verified ho

### Result

“Python exists but used nahi ho raha” wala doubt khatam.
