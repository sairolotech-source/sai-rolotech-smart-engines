Aapke screenshots me 3 alag signals dikh rahe hain:

**1. `FINAL: undefined`**

**2. `Δ [object Object]` warning**
Iska matlab UI me string ki jagah poora object directly print ho raha hai. Example:
`"Δ " + someObject`
to output `[object Object]` aata hai. Ye clean data formatting failure hai.

**3. React #185 crash in setup view**
Iska matlab Setup screen me koi `useEffect`, derived state, store subscription, ya computed setter loop me chal raha hai.

### Iska simple matlab

### Most likely root causes

Aapke audit aur screenshots ko mila ke, sabse possible causes ye hain:

**A. `useEffect` loop**

* `useEffect(() => setState(...), [thatSameState])`
* ya derived object har render me naya ban raha hai
* phir effect dobara chalta hai
* phir setState
* phir render
* phir crash

**B. Zustand/store selector unstable**
Agar aap store se object le rahe ho aur render ke andar usko reshape karke wapas set kar rahe ho, to loop ban sakta hai.

**C. Computed panel me invalid fallback**
`finalScore`, `delta`, `powerPattern`, `warnings` jaise fields undefined/object mix ke saath render ho rahe hain.

**D. Error boundary symptom hide kar rahi hai**
“Auto Recovery” screen aa rahi hai, but actual broken component same state ke saath dobara mount ho raha hoga.

### Abhi sabse pehle kya fix karna hai

Is order me karo:

**1. Setup view ka infinite loop pakdo**
Har `useEffect`, `useMemo`, `setState`, store update inspect karo.
Specially:

* setup screen
* power pattern
* warning banner
* final score calculation
* auto recovery mount flow

**2. `undefined` aur object rendering guard karo**
UI me raw object kabhi direct render mat karo.
Examples:

* `FINAL: {value ?? "N/A"}`
* `deltaLabel = typeof delta === "number" ? delta.toFixed(2) : "N/A"`
* object ko ya to serialize karo, ya exact field show karo

**3. One-way data flow banao**
Raw input → normalized data → computed results → readonly UI
UI render ke andar recompute + write-back mat karo.

**4. Setup screen me all setter calls isolate karo**
Render phase ke andar, conditional branch ke andar, ya unstable effect me `setState` mat chalao.

**5. Warning payload sanitize karo**
`Δ [object Object]` ka matlab warning formatter broken hai.
Expected:

* `Δ 1.24 mm`
* `Δ angle mismatch`
* `Δ gap unsafe`
  Not:
* `Δ [object Object]`
