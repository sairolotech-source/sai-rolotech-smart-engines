# Phase 1 — Crash source pakdo

## 1. Setup view ke sab `useEffect` nikaalo

Har file me search karo:

* `useEffect(`
* `setState(`
* `store.setState`
* `update...`
* `set...`

Special focus:

* `SetupView`
* `PowerPattern`
* `WarningBanner`
* `FinalScore/Result panel`
* `AutoRecovery/ErrorBoundary reset flow`

## 2. Ye dangerous pattern dhundo

Agar aisa kuch mile to almost pakka loop hai:

```ts
useEffect(() => {
  setX(computeFrom(x));
}, [x]);
```

ya

```ts
useEffect(() => {
  setForm({...form, final: calc(form)});
}, [form]);
```

ya

```ts
const derived = items.map(...)
useEffect(() => {
  setItems(derived)
}, [derived])
```

ya

```ts
if (something) {
  setState(...)
}
```

render body ke andar.

## 3. Temporary trace lagao

Har suspicious effect ke start me ye lagao:

```ts
useEffect(() => {
  console.log("EFFECT setupView fired", {
    thicknessBandMin,
    thicknessBandMax,
    profileSourceType,
    finalValue,
  });
}, [thicknessBandMin, thicknessBandMax, profileSourceType, finalValue]);
```

Har setter ke just pehle:

```ts
console.log("SET finalValue", nextFinalValue);
```

Agar same log 20-50 baar fast aa raha hai, wahi loop source hai.

# Phase 2 — Render-time writes hatao

## 4. Render ke andar koi setter nahi hona chahiye

Ye galat hai:

```ts
if (!finalValue) {
  setFinalValue(calc(...));
}
```

Ye bhi galat hai:

```ts
const result = calc(data);
setStoreResult(result);
return <UI />
```

Isko split karo:

* render me sirf read
* compute `useMemo` me
* write sirf user action ya controlled effect me

## 5. One-way flow enforce karo

Flow aisa hona chahiye:

**raw input**
→ **normalized input**
→ **computed result**
→ **UI render**

Na ki:
UI render → recompute → store write → rerender → crash

# Phase 3 — `FINAL: undefined` fix

## 6. Final result ka exact source dhundo

Search:

* `FINAL`
* `finalValue`
* `finalScore`
* `finalResult`
* `result.final`

Phir check karo:

* kya backend se field aa hi nahi rahi?
* field ka naam mismatch hai?
* nested object path galat hai?
* calc function `return` nahi kar raha?
* async load se pehle render ho raha hai?

## 7. Strict fallback lagao

UI me:

```ts
const finalLabel =
  typeof finalValue === "number"
    ? finalValue.toFixed(2)
    : "N/A";
```

Render:

```tsx
<div>FINAL: {finalLabel}</div>
```

Agar object ho sakta hai:

```ts
const finalValue =
  typeof result === "object" && result !== null && typeof result.final === "number"
    ? result.final
    : null;
```

## 8. Calc function ko harden karo

Example:

```ts
export function computeFinal(input: EngineInput): number | null {
  if (!input || !input.material || !input.thickness) return null;

  const value = /* calc */;
  return Number.isFinite(value) ? value : null;
}
```

# Phase 4 — `Δ [object Object]` fix

## 9. Warning formatter dhundo

Search:

* `Δ`
* `delta`
* `warning`
* `warnings.map`
* `label:`

Bug likely aisa hoga:

```ts
`Δ ${delta}`
```

jab `delta` actually object hai.

## 10. Scalar formatter banao

Example:

```ts
function formatDelta(delta: unknown): string {
  if (typeof delta === "number") return `Δ ${delta.toFixed(2)}`;
  if (
    typeof delta === "object" &&
    delta !== null &&
    "value" in delta &&
    typeof (delta as any).value === "number"
  ) {
    return `Δ ${(delta as any).value.toFixed(2)}`;
  }
  return "Δ N/A";
}
```

UI:

```tsx
<div>{formatDelta(warning.delta)}</div>
```

## 11. Raw object render block karo

Kabhi aisa mat karo:

```tsx
<div>{warning}</div>
<div>{delta}</div>
<div>{result}</div>
```

Agar debug ke liye dikhana hi hai:

```tsx
<pre>{JSON.stringify(result, null, 2)}</pre>
```

# Phase 5 — Zustand/store loop check

## 12. Selector instability check karo

Ye risky hai:

```ts
const derived = useStore(s => ({
  a: s.a,
  b: s.b,
  c: heavyCalc(s),
}));
```

Har render naya object banega.

Better:

```ts
const a = useStore(s => s.a);
const b = useStore(s => s.b);
const c = useMemo(() => heavyCalc({ a, b }), [a, b]);
```

## 13. Store me write-back cycle dhundo

Aisa kuch dangerous hai:

```ts
const result = useStore(s => s.result);

useEffect(() => {
  setResult(recompute(result));
}, [result]);
```

Ye direct loop source ho sakta hai.

## 14. Equality guards lagao

Store update se pehle compare karo:

```ts
setState(prev => {
  if (prev.finalValue === nextFinalValue) return prev;
  return { ...prev, finalValue: nextFinalValue };
});
```

Objects ke liye shallow/deep compare use karo.
