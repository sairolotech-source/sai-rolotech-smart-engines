# FINAL MASTER BLUEPRINT

## Roll Forming Auto + Semi Auto Engineering App

## 1. Main goal

App ka purpose:

* DXF drawing import kare
* geometry parse kare
* width / height / bend detect kare
* web / flange / lip detect kare
* profile type classify kare
* thickness aur material le
* flower pattern generate kare
* station count estimate kare
* shaft / bearing suggest kare
* roll OD / gap / clearance estimate kare
* machine layout suggest kare
* final engineering report banaye
* PDF export kare
* debug + test system ho
* Auto / Semi Auto / Manual review logic ho

---

## 2. Most important truth

**100% blind auto allowed nahi hona chahiye.**
System ka target hona chahiye:

* **100% safe flow**
* **high reliability**
* **fake output zero**
* **uncertain case me semi-auto**
* **contradiction case me manual review**

---

## 3. Final mode logic

### Auto Mode

Sirf tab allow ho jab:

* all critical engines pass
* confidence high ho
* no blocking contradiction

### Semi Auto Mode

Tab use ho jab:

* result mostly sahi ho
* but kuch fields confirm karni ho

User confirm kare:

* width
* height
* bend count
* profile type
* return bends
* material
* thickness
* stations
* shaft
* bearing

### Manual Review

Tab use ho jab:

* contradiction ho
* geometry dirty ho
* confidence low ho
* profile complex ho
* consistency engine fail ho

## 6. Confidence model

Total confidence = 100

* import: 20
* geometry: 20
* bend detection: 20
* section features: 15
* flower logic: 10
* station logic: 10
* mechanical: 5

### Final rules

* **85+** → Auto
* **65 to 84** → Semi Auto
* **below 65** → Manual review
* **any blocking contradiction** → Manual review

---

## 7. Every engine must return these fields

```json
{
  "engine": "engine_name",
  "status": "pass",
  "confidence": "high",
  "blocking": false,
  "warnings": [],
  "assumptions": []
}
```

Allowed values:

* status: `pass`, `fail`, `review_required`
* confidence: `high`, `medium`, `low`
* blocking: `true`, `false`

---

## 8. Non-negotiable rules

1. Fake result mat do
2. Missing data ke baad next stage mat chalao
3. Contradiction ho to block karo
4. Confidence low ho to auto mat allow karo
5. User-confirmed semi-auto data ko separate mark karo
6. Final report ko “preliminary engineering report” hi bolo
7. Final tooling-approved result ka jhootha claim mat karo
8. Har calculation ka basis visible ho
9. Debug output hamesha available ho
10. Test suite ke bina success mat dikhana

---

## 9. Consistency checks

System ko ye conflicts pakadne chahiye:

* lipped_channel but lips = 0
* simple_channel but return_bends > 0
* shutter_profile but station count too low
* bend_count high but station count too low
* heavy duty but shaft too small
* roll OD too close to shaft diameter
* bend_count = 0
* width/height invalid
* unsupported DXF too much

Agar issue blocking ho → manual review

---

## 10. Auto Mode rules

Auto Mode tabhi allow:

* import high confidence
* geometry high confidence
* bend detection high confidence
* section features high confidence
* no blocking consistency issue
* final confidence >= 85

Otherwise:

* auto stop
* semi-auto ya manual

## 11. Semi Auto rules

* bend count
* profile type

* material
* thickness
* station count
* shaft diameter
* bearing type

User correction ke baad pipeline recalculate karo.

Return:

* `mode = semi_auto_confirmed`

## 12. Manual review rules

Manual review तब force karo jab:

* contradictions ho
* confidence low ho
* complex section ho
* DXF unreliable ho
* bend detection uncertain ho
* lip detection uncertain ho
