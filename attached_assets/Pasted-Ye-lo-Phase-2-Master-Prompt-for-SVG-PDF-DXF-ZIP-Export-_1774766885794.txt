## FEATURE 4 — DXF OUTLINE EXPORT

Add support for DXF export.

### Minimum DXF deliverables:

For each station export:

1. groove/profile outline
2. strip/profile reference centerline if available
3. optional roll body section outline
4. optional dimension layer placeholders

### DXF goals:

* CAD-import friendly
* simple clean geometry
* no excessive fragmentation
* no random splines unless necessary
* use lines/arcs where possible
* consistent units in mm
* origin placement defined
* layer naming structured

### Suggested DXF layers:

* `OUTLINE`
* `CENTERLINE`
* `ROLL_PROFILE`
* `STRIP`
* `ANNOTATION`
* `CONSTRUCTION`

### Must ensure:

* DXF opens in common CAD tools
* no malformed entities
* no unit ambiguity
* no inverted geometry
* no exploded nonsense geometry from bad conversion

### Important:

If true production-grade DXF with dimensions is not yet stable, then first export **clean profile geometry DXF only**, and do not fake full drafting support.

Add a toggle:

**Manufacturing Release Mode**

When ON:

* export uses locked validated values only
* title block includes release state
* revision number required
* approval fields shown
* warning banner hidden from final drawing
* extra metadata included in title block

### Add title block fields:

* Revision
* Checked By
* Approved By
* Release Status
* Manufacturing Note
* Tolerance Note
* Scale
* Unit = mm

### Release states:

* Draft
* Internal Review
* Shop Drawing
* Manufacturing Release

### Rules:

* “Manufacturing Release” cannot export if required fields missing
* if bore/keyway/shaft/OD critical fields are missing, block release export
* normal draft export may still be allowed

## FEATURE 6 — REVISION CONTROL IN TITLE BLOCK

Add revision/version handling.

### Required:

* revision text like `R0`, `R1`, `R2`
* generated date
* optional last modified timestamp
* drawing number version suffix

### Example:

* Drawing No.: `SRT-ROLL-U-170x50-ST03-R1`
* Revision: `R1`

### Rules:

* station export must reflect current revision
* rerun with modified geometry should not silently reuse old revision text
* exports should be deterministic for same data + same revision
