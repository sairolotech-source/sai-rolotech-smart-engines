100% guarantee main honestly nahi de sakta, kyunki engineering auto-design me **wrong DXF, missing geometry, unsupported entity, material variation, springback, tooling assumptions** ki wajah se absolute 100% possible nahi hota.
Lekin **near-zero-error system** banaya ja sakta hai jisme **Auto Mode sirf tab chale jab sab checks pass hon**, aur warna system **Semi Auto Mode me safely fallback** ho.

Aapke liye best solution ye hai:

# ACCURACY CONTROL SYSTEM

App me 3 mode rakho:

## 1. Auto Mode

Sirf tab run ho jab:

* DXF parse pass
* geometry clean pass
* bend detection confidence high
* web/flange/lip detection pass
* material selected
* thickness valid
* station logic confidence acceptable
* no critical warning

Agar inme se ek bhi fail ho → Auto Mode band, Semi Auto Mode par bhejo.

## 2. Semi Auto Mode

System ye auto kare:

* DXF import
* width/height detect
* probable bends detect
* probable profile type suggest
* flower pattern suggest
* station range suggest

User confirm kare:

* bend count
* flange/web/lip
* profile type
* material
* thickness
* return bends
* selected station count

Is mode me risk bahut kam ho jayega.

## 3. Manual Verified Mode

Critical projects ke liye:

* user ya engineer har stage verify kare
* final roll/design report tabhi generate ho

---

# GOLDEN RULE

**Jahan confidence low ho, wahan output mat do.**
App ko bolna hai:

* “I am not confident”
* “Manual confirmation required”
* “Semi-auto correction needed”

Fake result kabhi nahi dena.

---

# ACCURACY GATE SYSTEM

Har engine ke saath 3 cheezein return honi chahiye:

* `status`: pass / fail / review_required
* `confidence`: high / medium / low
* `blocking`: true / false

Example:

```json
{
  "engine": "bend_detection_engine",
  "status": "review_required",
  "confidence": "low",
  "blocking": true,
  "reason": "Disconnected geometry caused uncertain bend detection"
}
```

Agar `blocking: true` ho to:

* Auto Mode stop
* Semi Auto open

---

# CRITICAL VALIDATION RULES

## Auto Mode tabhi allow karo jab:

* imported geometry supported ho
* entity count reasonable ho
* open contour usable ho
* duplicate lines clean ho
* bbox valid ho
* bend count confidence high ho
* section feature detection stable ho
* section type contradiction na ho
* thickness/material valid ho
* station count rule-based ho
* shaft/bearing result explainable ho

## Auto Mode turant reject karo agar:

* bend count 0 aaye
* width/height 0 aaye
* unsupported entities zyada ho
* geometry disconnected ho
* section type uncertain ho
* 2 engines contradictory result dein
* same profile ko alag logic alag output de raha ho

---

# CROSS-CHECK ENGINE

Ek naya engine banao:

## `consistency_engine`

Ye verify kare:

* bend count vs section type match?
* lipped_channel but lips = 0? fail
* simple channel but return bends > 0? review
* shutter profile but stations too low? fail
* heavy duty but shaft too small? fail
* roll OD too small vs shaft? fail

Example:

```json
{
  "engine": "consistency_engine",
  "status": "fail",
  "blocking": true,
  "reason": "Profile classified as lipped_channel but no lips detected"
}
```

Ye engine accuracy bahut badhata hai.

# CONFIDENCE SCORING SYSTEM

Har result ko score do:

* Import confidence: /20
* Geometry confidence: /20
* Bend confidence: /20
* Section feature confidence: /15
* Flower logic confidence: /10
* Station logic confidence: /10
* Mechanical confidence: /5

Total = 100

Rule:

* 85+ → Auto Mode allowed
* 65–84 → Semi Auto only
* below 65 → Manual review only
