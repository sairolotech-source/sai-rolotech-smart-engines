==================================================
4. MANDATORY PIPELINE STAGES
============================

STAGE A — INPUT INGESTION
Accepted inputs:

* DXF profile
* center-line geometry
* manually drawn profile
* thickness
* material
* bend preferences
* station limit

Validate:

* file parsed correctly
* entities supported
* no empty geometry
* no broken polyline chain
* no invalid arc data
* unit detection clear
* thickness > 0
* profile not self-contradictory

Output:

* raw geometry object
* parse warnings
* parse errors

---

STAGE B — GEOMETRY NORMALIZATION
Tasks:

* merge connected entities
* sort segment order
* repair tiny gaps within tolerance
* remove duplicates
* unify units
* detect open vs closed shape
* detect arcs / lines / corners
* create canonical profile structure

Output:

* normalized profile
* ordered segments
* normalized coordinate system

---

STAGE C — CENTER-LINE TO SHEET CONVERSION
This is mandatory.

If input is center-line:

* convert center-line to sheet profile using thickness logic
* support both-side offset where applicable
* support explicit inside/outside/neutral-axis mode
* handle corner joins safely
* prevent self-intersection
* trim/extend intersections correctly
* preserve manufacturable geometry

Required options:

* centerline offset mode
* thickness application mode
* bend radius handling
* acute corner handling

Output:

* valid sheet profile
* inner/outer/neutral path references
* conversion warnings if geometry is unsafe

---

STAGE D — DIMENSION & ANGLE EXTRACTION
Mandatory outputs:

* segment lengths
* total profile width/height
* bend angles
* radii
* flange lengths
* coordinate references

Rules:

* dimensions must be visible and inspectable
* angles must not be blank
* any uncertain angle must be flagged, not guessed silently

---

STAGE E — FLOWER PATTERN GENERATION
Generate forming sequence from final section to intermediate passes.

Rules:

* use validated bend count only
* support multi-bend profiles
* maintain progressive forming logic
* enforce station count constraints
* if requested stations are fewer than ideal, produce controlled redistribution warning
* no random interpolation without report

Output:

* pass-by-pass intermediate profiles
* forming percentage per stand
* warnings for over-aggressive forming jumps

---

STAGE F — PASS / SMART ROLL DATA GENERATION
Before rendering, generate a complete pass object.

Each pass object must contain validated fields like:

* id
* geometry
* passLineY
* pass center reference
* width
* bend progression
* upper/lower roll references if applicable

If any required field is missing:

* mark pass invalid
* stop render
* show structured diagnostic panel

Do not allow UI to access undefined passLineY.

---

STAGE G — ROLL TOOLING CALCULATION
Compute:

* roll positions
* upper/lower clearances
* shaft constraints
* basic interference risk
* estimated OD references where rules allow
* stand sequence data

Rules:

* if machine constraints missing, state assumptions clearly
* if tooling is approximate, label as approximation
* never label estimated tooling as production released unless verified

---

STAGE H — RENDERING
Render only verified data.

Before rendering each module:

* Setup View
* Roll View
* Smart-Rolls
* Flower View
* Tooling View

Use a pre-render validator.

If invalid:

* do not crash
* render diagnostic state
* show missing field names
* offer retry only after data refresh
