## Codex ke liye full master prompt

You are the lead engineer for a production-grade Roll Forming + Tooling + CNC software system.

You are NOT allowed to give fake completion messages.
You must inspect the full repository, identify all broken or half-built systems, implement missing engines, connect them end-to-end, run validation, and leave the app in a working state.

==================================================

1. PRODUCT INTENT
   ==================================================

This app is a roll forming design and tooling generation system.

The user should be able to:

1. Upload or define a profile
2. Enter material thickness
3. Generate flower pattern / station progression
4. Generate per-station roll tooling geometry
5. Generate upper and lower roll groove coordinates
6. Compute roll dimensions
7. View station-by-station status
8. Export DXF / CSV / XML
9. Export CNC lathe G-code
10. See dimensions, bend angles, and warnings
11. Recover old saved jobs from localStorage without corruption
12. Avoid crashes even when partial data exists

This is not a demo app.
This must behave like a real engineering workflow.

The app is partially built.
Replit previously created UI guards and some status logic, but the real engineering engines are incomplete.

Current known problems include:

C. Stations are showing BASIC instead of VALID
D. upperRoll[] is empty
E. lowerRoll[] is empty or incomplete

K. Thickness-driven profile conversion is incomplete
L. Flower pattern generation is incomplete
M. Auto dimensions / bend angles are incomplete or missing
N. Some features claim completion without actual working output

Do not hide any of this.
Fix it.

Do not patch only the UI.

Do not weaken validation.
Do not silently skip important engineering failures.

You must make the system genuinely functional.

Audit the codebase and organize it into these logical engines if missing:

==================================================
5. PROFILE INPUT / IMPORT ENGINE
================================

The app must support profile input from either:

* manual drawing / internal profile points
* DXF upload
* centerline-based sketch converted to sheet profile

Required behavior:

A. DXF upload must parse geometry into usable profile entities
B. The profile must preserve:

* lines
* arcs
* bend transitions where possible
  C. The app must detect:
* overall width
* leg lengths
* bend points
* angle transitions
  D. If profile is centerline-based, the app must convert it into sheet profile using material thickness
  E. The app must explicitly ask or use:
* thickness
* profile reference mode
* dimension basis
  F. If the profile cannot be resolved, show exact reason

No fake success state.

==================================================
6. CENTERLINE TO SHEET ENGINE
=============================

This is mandatory.

If a user draws centerline geometry and provides thickness,
the app must generate sheet boundaries correctly.

Rules:

1. For straight segments:

   * offset equally around centerline when appropriate
2. For bends:

   * compute inner / outer / neutral path relationship
3. Thickness must affect resulting profile
4. Conversion must work for common roll-formed sections:

   * C
   * U
   * Z
   * hat / channel
   * simple shutter-type profiles
5. Result must produce actual sheet edges, not only display lines

Expected outputs:

* innerProfilePoints
* outerProfilePoints
* centerlineProfilePoints
* developedWidth estimate
* segment metadata

==================================================
7. FLOWER PATTERN ENGINE
========================

This engine is mandatory and currently incomplete.

The app must generate station progression from input profile.

Required behavior:

1. Analyze target profile bend sequence
2. Split forming into stations
3. Generate progressive bend state per station
4. Support at minimum:

   * simple channels
   * C/U sections
   * Z sections
   * multi-flange sections
5. Each station must contain:

   * intermediate section points
   * bend progress
   * angle state
   * notes / warnings
6. If the final profile implies more stations than requested:

   * show warning
   * allow compressed pass schedule
   * mark risk level
7. Do not output nonsense flower diagrams

Outputs required:

* stationSections[]
* stationAngles[]
* stationMetadata[]
* warnings[]

==================================================
8. STATION SYNTHESIS ENGINE
===========================

Each station must become a real engineering object.

For every station create:

* stationIndex
* formedSectionPoints
* passLineY
* materialThickness
* bendSummary
* rollProfile
* validationStatus
* diagnostics

If geometry exists, station cannot stay empty.

==================================================
9. TOOLING ENGINE
=================

This is the main missing engine.
Implement fully.

Create or repair toolingEngine.ts

Required functions:

* synthesizeGroove()
* buildStationRollProfile()
* buildAllStationTooling()

Each station must generate:

* upperRoll: Point2D[]
* lowerRoll: Point2D[]
* upperRollDiameter
* lowerRollDiameter
* shaftDiameter
* grooveDepth
* faceWidth
* passLineY
* notes[]

Rules:

1. Groove must be derived from station section
2. Thickness must influence groove shape
3. Upper and lower rolls must be complementary
4. Small configurable clearance must exist
5. No empty arrays
6. No placeholder data
7. No silent failure

If a station has enough section geometry, rollProfile must be synthesized.

==================================================
10. ROLL GEOMETRY ENGINE
========================

Add real roll-related calculations.

At minimum calculate:

* roll diameter
* shaft diameter
* face width
* groove depth
* vertical relation to pass line
* upper/lower roll gap
* interference risk flag

If exact manufacturing logic is unavailable, use clear engineering approximations and mark assumptions.

But do not leave values undefined.

==================================================
11. CNC LATHE ENGINE
====================

Create or repair cncLatheEngine / gcodeLathe.ts

Required function:

* generateLatheGcode()

For both upper and lower rolls generate real G-code strings.

Required output characteristics:

* G21
* G18
* safe start
* spindle start
* X/Z positioning
* profile path feed moves
* safe retract
* stop
* M30

Do not leave:

* upperLatheGcode = ""
* lowerLatheGcode = ""

If geometry exists, G-code must be generated.

13. VALIDATION ENGINE
=====================

Validation must be strict.

Statuses:

* VALID
* BASIC
* MISSING_PROFILE
* INVALID_GEOMETRY
* EXPORT_BLOCKED

Rules:

VALID only if:

* rollProfile exists
* upperRoll has > 1 point
* lowerRoll has > 1 point
* upperLatheGcode length > 20
* lowerLatheGcode length > 20

BASIC if:

* rollProfile exists
* but groove data or G-code is incomplete

MISSING_PROFILE if:

* no rollProfile exists

INVALID_GEOMETRY if:

* section points are missing or invalid

Return diagnostic reason text for every station.

==========================================

Repair old saved projects fully.

Create or repair migrationRepairEngine and migrate() logic.

Required behavior:

* detect stale versions

Do not only update schema numbers.

==================================================
15. EXPORT ENGINE
=================

Exports must reflect actual data truthfully.

A. DXF export

* export geometry only when real geometry exists
* include station-related profile data if supported

B. CSV export

* include per-station values

C. XML export

Rules:

* VALID: allow all exports
* BASIC: allow geometry exports only if data exists; block CNC export with exact reason
* MISSING_PROFILE / INVALID_GEOMETRY: skip with console.warn and visible UI status

No silent skip.
