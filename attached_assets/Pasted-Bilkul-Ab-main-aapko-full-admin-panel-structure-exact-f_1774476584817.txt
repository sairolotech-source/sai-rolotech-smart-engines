# 1) USER ROLES STRUCTURE

Software me alag-alag roles hone chahiye.

## A. Super Admin

Ye highest level access hoga.

### Permissions:

* sab users dekh sakta hai
* new users create kar sakta hai
* material library edit kar sakta hai
* machine library edit kar sakta hai
* profile templates add kar sakta hai
* system rules change kar sakta hai
* export settings control kar sakta hai
* full reports dekh sakta hai

---

## B. Admin

Ye company level admin hoga.

### Permissions:

* company ke users manage kare
* projects create/delete kare
* materials use kare
* machine templates assign kare
* exports dekh sake
* reports dekh sake
* engineer approvals monitor kare

---

## C. Design Engineer

Ye main working user hoga.

### Permissions:

* project create kare
* profile draw kare
* DXF import kare
* material select kare
* flower pattern generate kare
* roll draft generate kare
* warnings dekhe
* manual override kare
* export nikale

---

## D. Production Engineer

Ye shop floor ya setup check ke liye useful hoga.

### Permissions:

* approved designs dekhe
* tooling sheets dekhe
* setup sheets dekhe
* revision history dekhe
* shop feedback add kare

---

## E. Viewer / Client

Ye sirf dekh sake.

### Permissions:

* selected reports dekhe
* PDF download kare
* comments kar sake
* edit na kare

# 14) REPLIT KO DENE KE LIYE DEVELOPMENT PROMPT

```text
Build a modular industrial roll forming design software with a complete admin panel and engineer workflow.

The app must include:
1. Authentication and role-based access control
2. User roles: Super Admin, Admin, Design Engineer, Production Engineer, Viewer
3. Admin modules:
   - user management
   - material master
   - machine master
   - standard profile library
   - engineering rules manager
   - approval workflow
   - export manager
   - audit logs
4. Engineer modules:
   - create project
   - draw/import profile
   - select material
   - define min, nominal, and max thickness
   - select machine
   - analyze profile geometry
   - generate flower pattern
   - generate draft roll tooling
   - validate risks
   - export reports
5. Database must support:
   - users
   - projects
   - materials
   - profiles
   - profile_segments
   - machine_setups
   - flower_patterns
   - flower_pattern_stations
   - roll_designs
   - validations
   - exports
   - approvals
   - audit_logs
6. Include industrial UI with dashboard cards, engineering panels, warnings, and structured forms
7. Use React + TypeScript frontend and Node.js or Python backend
8. Use PostgreSQL with JSON fields for geometry and contour storage
9. Make the app scalable for future AI recommendation and CAD/simulation features
10. This is a semi-automatic engineering system, not a blind one-click final design tool
```
